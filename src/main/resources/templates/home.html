Tips

Open Ports
ORPort: This is the "ORPort" that your relay will use to communicate with other relays in the Tor network.
ListenAddrPort: This is the "ListenAddrPort" that your relay will use to accept incoming connections.
WebTunnel Ports: For WebTunnel, ports 80 and 443 need to be open.
Control Port: This is the port that Tor uses to listen for commands and events. It's used to control the Tor process.
Limits
Why Limits: Limits are set based on what Tor allows on one IP address. It's recommended to only run one type of relay at a time.
Guard Relays: The limit for guard relays is 8.
Obfs Bridge: The limit for obfs bridge is 2.
WebTunnel: The limit for WebTunnel is 1.
Snowflake Proxy: The limit for snowflake proxy is 1.
Onion Service: Onion service is without a number limit.
Page Information
Home Setup Page: User sets up new Tor services on this page.
Traffic Data: User can see traffic on his relays and events on this page.
Tor Proxy: This page is for setting up a proxy.
Relay Operations: User can see which relays are running, start and stop those relays, and edit them on this page.
Relay Types
Middle Relay: This is a type of relay that passes traffic within the Tor network. It doesn't know the source or destination of the traffic.
Guard Relay: This is the first relay in the Tor circuit. It knows the source of the traffic but not the destination.
Bridge: This is a type of relay that isn't listed in the public Tor directory, making it harder for censors to block.
Onion Service: This allows users to access services without revealing their location or the location of the service.
Choosing a Bridge
Obfs4: Use this when you have an open IP.
WebTunnel: Use this when you have an open IP and a domain.
Snowflake: Use this when you don't have either an open IP or a domain.
